Kalshi contract tickers encode all needed info:
  KXHIGHDEN-26JUN11-B78.5
  → series: KXHIGHDEN
  → station: KDEN (strip KX + HIGH/LOW + city code + add K prefix)
  → date: 26JUN11 = June 11, 2026
  → threshold: B78.5 = between bucket at 78.5°F

City code mapping:
  DEN → KDEN  (Denver)
  MIA → KMIA  (Miami)
  PHX → KPHX  (Phoenix)
  LAX → KLAX  (LA)
  DFW → KDFW  (Dallas)
  NYC → KNYC  (NYC)
  CHI → KORD  (Chicago)
  AUS → KAUS  (Austin)

TASK 2 — CREATE FAST KALSHI PRICE POLLER
================================================================

Create new file: scripts/trading/kalshi_price_poller.py

This is a standalone lightweight script:
  - Imports KalshiClient from kalshi_client.py
  - Runs in infinite loop
  - Every 5 seconds: calls get_weather_markets()
  - Writes to weather_contract_prices table
  - Logs summary every 60 seconds only (not every 5 — too noisy)
  - Handles all errors gracefully — never crashes
  - Reads env from /etc/bhn-trading/strat9.env + /etc/bhn-trading/env

Structure:
  def poll_once(client, conn):
      markets = client.get_weather_markets()
      # upsert to weather_contract_prices
      return len(markets)

  def main():
      client = KalshiClient(...)
      conn = get_pg_connection()
      while True:
          try:
              n = poll_once(client, conn)
              if time.time() - last_log > 60:
                  logger.info(f"Kalshi poller: {n} prices updated")
                  last_log = time.time()
          except Exception as e:
              logger.error(f"Poll error: {e}")
          time.sleep(5)

Create systemd service:
  /etc/systemd/system/bhn-kalshi-prices.service
  [Unit]
  Description=BHN Kalshi price poller (5 second interval)
  After=network.target postgresql.service

  [Service]
  Type=simple
  User=root
  WorkingDirectory=/opt/bhn/trading
  ExecStart=/usr/bin/python3 /opt/bhn/trading/kalshi_price_poller.py
  EnvironmentFile=/etc/bhn-trading/env
  EnvironmentFile=/etc/bhn-trading/strat9.env
  Restart=always
  RestartSec=10
  StandardOutput=journal
  StandardError=journal

  [Install]
  WantedBy=multi-user.target

NOTE: This is a SERVICE not a timer — runs continuously.
Enable: systemctl enable --now bhn-kalshi-prices.service
Verify: systemctl status bhn-kalshi-prices.service

================================================================

Dashboard panels:

PANEL 1 — Kalshi Live Prices (time series)
  Title: Live Kalshi Prices
  Query:
    SELECT captured_at, contract_title, yes_price
    FROM weather_contract_prices
    WHERE captured_at > NOW() - INTERVAL '2 hours'
    AND yes_price IS NOT NULL
    ORDER BY captured_at
  Visualization: Time series
  Refresh: 5s

PANEL 2 — BHN vs Market Edge (table)
  Title: BHN vs Market Edge
  Query:
    SELECT
      wcp.contract_title,
      wcp.contract_id AS ticker,
      ROUND(AVG(wf.predicted_value),1) AS nws_forecast_f,
      wcp.yes_price AS market_prob,
      wcp.captured_at
    FROM weather_contract_prices wcp
    LEFT JOIN prediction_contracts pc
      ON pc.station_code IS NOT NULL
      AND wcp.contract_id LIKE '%' ||
          REPLACE(pc.station_code,'K','') || '%'
    LEFT JOIN weather_forecasts wf
      ON wf.station_code = pc.station_code
      AND wf.source_model = 'nws_gridpoints'
      AND wf.target_date = pc.resolution_date
    WHERE wcp.captured_at > NOW() - INTERVAL '35 minutes'
    GROUP BY wcp.contract_title, wcp.contract_id,
             wcp.yes_price, wcp.captured_at
    ORDER BY wcp.contract_id
  Visualization: Table
  Refresh: 30s

PANEL 3 — Open Positions P&L (table)
  Title: My Kalshi Positions
  Query:
    SELECT contract_ticker, contract_title, side,
           num_contracts, avg_price_cents,
           cost_usd, market_value_usd,
           unrealized_pnl_usd, payout_if_right_usd,
           captured_at
    FROM kalshi_positions
    ORDER BY captured_at DESC
  Visualization: Table
  Refresh: 30s
  Note: Empty until kalshi_positions table approved + created

PANEL 4 — Data Freshness (stat panels)
  Title: Data Freshness
  Four stat panels:
    NWS Last Update:
      SELECT ROUND(EXTRACT(EPOCH FROM
        (NOW()-MAX(predicted_at)))/60,0) AS minutes_ago
      FROM weather_forecasts
      WHERE source_model='nws_gridpoints'

    Kalshi Last Update:
      SELECT ROUND(EXTRACT(EPOCH FROM
        (NOW()-MAX(captured_at)))/60,0) AS minutes_ago
      FROM weather_contract_prices

    ASOS Last Update:
      SELECT ROUND(EXTRACT(EPOCH FROM
        (NOW()-MAX(observed_at)))/60,0) AS minutes_ago
      FROM weather_observations

    Active Contracts:
      SELECT COUNT(*) FROM prediction_contracts
      WHERE is_active=true
  Visualization: Stat
  Refresh: 30s

PANEL 5 — Calibration Progress (bar gauge)
  Title: Days to Go-Live (need 30)
  Query:
    SELECT station_code,
           COUNT(DISTINCT target_date) AS days_collected
    FROM weather_forecasts
    WHERE source_model = 'nws_gridpoints'
    AND variable = 'tmax_f'
    GROUP BY station_code
    ORDER BY station_code
  Visualization: Bar gauge (max=30)
  Refresh: 1h

TASK 5 — UPDATE METABASE BHN vs MARKET EDGE QUERY
================================================================

After station_code is backfilled in prediction_contracts,
update the Metabase SQL file:
  infrastructure/docs/metabase/weather_forecast_vs_market.sql

New working query:
  SELECT
      wcp.contract_title,
      wcp.contract_id AS ticker,
      pc.station_code,
      pc.resolution_date,
      wcp.yes_price AS market_implied_prob,
      ROUND(AVG(wf.predicted_value), 1) AS nws_forecast_f,
      ROUND(AVG(wf.predicted_value) - (wcp.yes_price * 100), 1)
          AS edge_degrees,
      wcp.captured_at AS price_as_of
  FROM weather_contract_prices wcp
  JOIN prediction_contracts pc
      ON pc.station_code IS NOT NULL
      AND wcp.contract_id LIKE '%' ||
          REPLACE(pc.station_code,'K','') || '%'
  LEFT JOIN weather_forecasts wf
      ON wf.station_code = pc.station_code
      AND wf.source_model = 'nws_gridpoints'
      AND wf.target_date = pc.resolution_date
  WHERE wcp.captured_at > NOW() - INTERVAL '35 minutes'
  AND pc.is_active = true
  GROUP BY wcp.contract_title, wcp.contract_id,
           pc.station_code, pc.resolution_date,
           wcp.yes_price, wcp.captured_at
  ORDER BY pc.resolution_date, pc.station_code;
