SECTION 4 — ARCHITECTURE DECISIONS MADE THIS SESSION
================================================================================

CARD IDENTITY SYSTEM (implemented tonight by Claude Code):
  card_code  = BST-004-1E   (stored on master_card_catalog)
               Format: [SET_CODE]-[CARD_NUMBER_PADDED_3]-[EDITION_CODE]
               Print variant appended only when non-Standard (e.g. BST-058-1E-ERR)
  slab_code  = BST-004-1E-PSA-10   (derived, never stored)
               Format: [CARD_CODE]-[GRADER]-[GRADE_NUMERIC]
  card_id    = integer PK — always the join key, never card_code

SCHEMA RESTRUCTURE — APPROVED NAMING CONVENTION (not yet implemented):
  [market]_bids          — open buy orders / offers / OBO
  [market]_asks          — active + historical sell listings with outcome tracking
  [market]_transactions  — completed sales (immutable)

  Markets: ebay, courtyard, collector_crypt

  OLD → NEW:
    ebay_listings        → ebay_asks
    sold_listings        → ebay_transactions
    courtyard_listings   → courtyard_asks
    courtyard_sales      → courtyard_transactions
    collector_crypt_sales → collector_crypt_transactions

  NEW TABLES TO CREATE:
    ebay_bids, courtyard_bids, collector_crypt_bids
    collector_crypt_asks
    order_price_history  (bid + ask price changes, all markets)
    fee_schedule         (platform rate reference)
    arbitrage_positions  (full trade lifecycle + P&L)

  BACK-COMPAT VIEWS: maintain for all renames until n8n workflows migrated

THREE-WAY COST COMPARISON (approved, not yet implemented):
  MARKET   = published platform rate from fee_schedule
  ESTIMATED = pre-trade projection from estimate_trade_costs()
  ACTUAL   = what was actually charged post-trade

  All three tracked on arbitrage_positions per cost line item.
  Deltas: market→est, est→actual, market→actual
  Purpose: calibrate estimates over time, track promo savings

POSTGRESQL SCHEMA SEPARATION (deferred — do AFTER bids/asks/transactions rename):
  Proposed 4-schema structure:
    pokemon   — all card market tables

    horizon   — memories, chat, agent state
  Current: all tables in public schema
  Action: revisit after schema restructure is stable
