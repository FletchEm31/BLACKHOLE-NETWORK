WEATHER BETTING DATA ARCHITECTURE
Bronze / Silver / Gold Tables

GOAL
Build a clean pipeline for Kalshi weather markets using NWS forecasts, Open-Meteo forecasts, Kalshi market prices, and NWS actual settlement data.
The system should preserve raw data, clean and standardize it, then create model-ready features and edge outputs.

REFINED SILVER
============================================================

- silver_model_base

One clean row per city, date, run time, and contract side.

============================================================
BUILD ORDER
============================================================

1) Ingest raw API data into bronze tables.
2) Clean and standardize into silver tables.
3) Join forecasts, prices, and actuals into silver_model_base.
4) Train calibration on forecast error labels.
5) Generate calibrated probabilities.
6) Write edge outputs into gold tables.
7) Use gold_daily_edge_sheet for daily betting decisions.

============================================================
NOTES
============================================================

- Preserve raw API payloads in bronze tables.
- Do not mix forecast issue times with target dates.
- Keep high and low contract sides separate.
- Use actual - forecast for error labeling.
- Train calibration after enough matched history is collected.
- Keep source, timestamp, and station mapping explicit for every row.
