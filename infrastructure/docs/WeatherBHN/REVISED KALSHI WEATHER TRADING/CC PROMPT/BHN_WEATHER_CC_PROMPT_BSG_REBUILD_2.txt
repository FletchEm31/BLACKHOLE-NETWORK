TASK 7 — CREATE weather_gold_daily_edge_sheet POPULATION SCRIPT
================================================================

Create scripts/trading/weather_edge_calculator.py

This script:
1. Reads weather_silver_model_base for today + tomorrow
2. For each city/date/side:
   a. Gets latest NWS forecast from weather_silver_forecast_conformed
   b. Gets latest GFS forecast from weather_silver_forecast_conformed
   c. Computes model_delta_f
   d. Gets calibration bias from weather_silver_forecast_error
      (avg forecast_error last 30 days for this station)
   e. Applies bias correction to NWS forecast
   f. Computes raw bucket probability using normal distribution
   g. Applies calibration (isotonic if available, else passthrough)
   h. Gets latest Kalshi market price from weather_silver_market_conformed
   i. Computes edge = calibrated_prob - market_implied_prob
   j. Determines trade_flag:
      edge >= 0.05  → 'BET_YES'
      edge <= -0.05 → 'BET_NO'
      else          → 'SKIP'
   k. Determines model_confidence:
      model_delta_f < 2  → 'HIGH'
      model_delta_f < 4  → 'MEDIUM'
      else               → 'LOW'
   l. Computes Kelly fraction and stake_usd
   m. Writes to weather_gold_daily_edge_sheet

Run this script after each weather collection cycle:
  Add to weather_data_collector.py as last step in main()

-- QUERY 1: Daily Edge Sheet (THE MAIN VIEW)
-- Shows exactly what to trade today and tomorrow
SELECT
    city,
    station_code,
    target_date,
    contract_side,
    bucket_label,
    contract_ticker,
    raw_forecast_f        AS nws_forecast,
    gfs_forecast_f        AS gfs_forecast,
    model_delta_f         AS model_delta,
    model_confidence,
    ROUND(calibrated_prob * 100, 1) AS model_prob_pct,
    ROUND(market_implied_prob * 100, 1) AS market_prob_pct,
    ROUND(edge_pct, 1)    AS edge_pct,
    market_volume,
    market_liquidity,
    recommended_action,
    stake_usd,
    skip_reason,
    last_updated
FROM weather_gold_daily_edge_sheet
WHERE target_date BETWEEN CURRENT_DATE AND CURRENT_DATE + 1
AND is_active = TRUE
ORDER BY target_date, edge DESC;

-- QUERY 2: Latest NWS vs GFS Forecasts
SELECT
    city,
    station_code,
    target_date,
    source_name,
    tmax_f,
    tmin_f,
    cloud_cover,
    pop,
    wind_speed,
    is_latest_run,
    forecast_run_time
FROM weather_silver_forecast_conformed
WHERE target_date BETWEEN CURRENT_DATE AND CURRENT_DATE + 2
AND is_latest_run = TRUE
ORDER BY target_date, station_code, source_name;

-- QUERY 3: Live Kalshi Market Prices
SELECT
    city,
    station_code,
    contract_side,
    bucket_label,
    market_ticker,
    target_date,
    ROUND(implied_prob * 100, 1) AS implied_prob_pct,
    yes_bid,
    yes_ask,
    yes_mid,
    volume,
    market_liquidity_flag,
    snapshot_time
FROM weather_silver_market_conformed
WHERE is_latest_snapshot = TRUE
AND target_date BETWEEN CURRENT_DATE AND CURRENT_DATE + 1
ORDER BY target_date, station_code, contract_side, bucket_floor;

-- QUERY 4: Forecast Accuracy (NWS Error by City)
SELECT
    city,
    station_code,
    feature_name,
    source_name,
    COUNT(*) AS sample_size,
    ROUND(AVG(forecast_error_f), 2) AS avg_bias_f,
    ROUND(AVG(ABS(forecast_error_f)), 2) AS mae_f,
    ROUND(STDDEV(forecast_error_f), 2) AS std_f,
    SUM(CASE WHEN error_sign = 'hot' THEN 1 ELSE 0 END) AS times_ran_hot,
    SUM(CASE WHEN error_sign = 'cold' THEN 1 ELSE 0 END) AS times_ran_cold
FROM weather_silver_forecast_error
WHERE target_date > CURRENT_DATE - 30
GROUP BY city, station_code, feature_name, source_name
ORDER BY station_code, feature_name, source_name;

-- QUERY 5: Calibration Progress
SELECT
    sfc.station_code,
    sfc.city,
    sfc.contract_side,
    COUNT(DISTINCT sfc.target_date) AS forecast_days,
    COUNT(DISTINCT sa.target_date) AS settled_days,
    COUNT(DISTINCT sfe.target_date) AS error_pairs,
    CASE
        WHEN COUNT(DISTINCT sfe.target_date) >= 30
        THEN 'READY TO CALIBRATE'
        ELSE CONCAT(
            CAST(30 - COUNT(DISTINCT sfe.target_date) AS TEXT),
            ' days remaining'
        )
    END AS calibration_status,
    MIN(sfc.target_date) AS earliest_forecast,
    MAX(sa.target_date) AS latest_settlement
FROM weather_silver_forecast_conformed sfc
LEFT JOIN weather_silver_actuals_conformed sa
    ON sa.station_code = sfc.station_code
    AND sa.target_date = sfc.target_date
LEFT JOIN weather_silver_forecast_error sfe
    ON sfe.station_code = sfc.station_code
    AND sfe.target_date = sfc.target_date
WHERE sfc.is_latest_run = TRUE
AND sfc.source_name = 'nws'
GROUP BY sfc.station_code, sfc.city, sfc.contract_side
ORDER BY sfc.station_code, sfc.contract_side;

-- QUERY 6: Data Freshness
SELECT
    source_label,
    station_code,
    MAX(last_update) AS last_update,
    ROUND(EXTRACT(EPOCH FROM (NOW() - MAX(last_update)))/60, 0)
        AS minutes_ago
FROM (
    SELECT 'NWS Forecast' AS source_label,
           station_code,
           MAX(retrieved_at) AS last_update
    FROM weather_bronze_nws_forecast_snapshots
    GROUP BY station_code

    UNION ALL
    SELECT 'Open-Meteo GFS',
           station_code,
           MAX(retrieved_at)
    FROM weather_bronze_openmeteo_forecast_snapshots
    WHERE model = 'gfs_seamless'
    GROUP BY station_code

    UNION ALL
    SELECT 'Kalshi Prices',
           station_code,
           MAX(retrieved_at)
    FROM weather_bronze_kalshi_market_snapshots
    GROUP BY station_code

    UNION ALL
    SELECT 'NWS Actuals',
           station_code,
           MAX(retrieved_at)
    FROM weather_bronze_nws_actuals
    GROUP BY station_code
) t
GROUP BY source_label, station_code
ORDER BY source_label, station_code;

TASK 11: Visual Crossing Historical Actuals Backfill
================================================================

GOAL
----
Backfill 160+ days of historical daily tmax/tmin into the bronze and silver
layers so weather_silver_forecast_error has enough rows (≥30 per station) to
switch from hardcoded SIGMA_DEFAULTS to computed MAE in the gold edge calculator.

Without this, all 8 cities use hardcoded sigma fallbacks indefinitely (requires
30 days of natural NWS CLI accumulation). With this backfill, real calibrated
sigma is available on the FIRST run after completing the task.

PREREQ
------
1. Sign up at https://www.visualcrossing.com (free tier: 1000 records/day;
   one per-city date-range request returns up to N days — 8 cities × 1 request
   = 8 API calls, each returning ~160 rows. Free tier covers this in one session.)
2. Add API key to /etc/bhn-trading/strat9.env on LA server:
      VISUAL_CROSSING_API_KEY=<your_key>

SCRIPT
------
  scripts/trading/weather_vc_backfill.py

WHAT IT DOES
------------
- Calls VC timeline API per city:
    GET https://weather.visualcrossing.com/.../timeline/{ICAO}/{start}/{end}
    ?unitGroup=us&elements=datetime,tempmax,tempmin&include=days&key={KEY}
- Date range: 2026-01-01 → yesterday (default; overridable via --start/--end)
- Writes to weather_bronze_nws_actuals with source_payload_json.source =
  "visual_crossing_backfill". ON CONFLICT DO NOTHING preserves existing NWS rows.
- Writes to weather_silver_actuals_conformed with actual_source = 'visual_crossing'
  (separate rows from 'nws_cli' — both contribute to forecast_error).
- For each silver row inserted, joins weather_silver_forecast_conformed to populate
  weather_silver_forecast_error pairs automatically (same logic as NWS path).

USAGE
-----
  # Dry-run first to verify connectivity:
  python3 weather_vc_backfill.py --dry-run

  # Full backfill:
  python3 weather_vc_backfill.py

  # Custom range:
  python3 weather_vc_backfill.py --start 2025-10-01 --end 2026-06-10

EXPECTED OUTCOME
----------------
  SELECT station_code, COUNT(*) FROM weather_bronze_nws_actuals
  GROUP BY 1 ORDER BY 1;
  → ~160+ rows per station (down from 1 row)

  SELECT station_code, COUNT(*) FROM weather_silver_forecast_error
  GROUP BY 1 ORDER BY 1;
  → Non-zero rows (depends on how many forecast_conformed rows exist for those dates)

  After running weather_edge_calculator.py:
  → sigma column in edge sheet switches from SIGMA_DEFAULTS to computed MAE
    for any station with ≥30 rows in weather_silver_forecast_error

- bronze actuals row count per station
- silver_forecast_error row count per station
- Whether edge calc switched to computed sigma (check logs for "using computed MAE")
