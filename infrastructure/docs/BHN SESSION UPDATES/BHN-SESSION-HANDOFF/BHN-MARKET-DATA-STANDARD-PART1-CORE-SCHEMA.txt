NAMING CONVENTION:
  [market]_bids           — open buy orders / offers / OBO
  [market]_asks           — sell listings (active + historical with outcome)
  [market]_transactions   — completed sales (immutable)

MARKETS:
  ebay                    — traditional marketplace (eBay.com)
  courtyard               — tokenized NFTs on Polygon (Courtyard.io via OpenSea)
  collector_crypt          — tokenized NFTs on Solana (Collector Crypt via Magic Eden)

DATA SOURCES (verified):
  Courtyard asks/bids:     OpenSea v2 API (Polygon, free key required)
  Courtyard transactions:  OpenSea v2 API events endpoint
  Collector Crypt asks:    Magic Eden Solana API (free, 120 QPM)
  Collector Crypt bids:    Magic Eden bids endpoint (verified)
  Collector Crypt txns:    Magic Eden activity endpoint
  eBay asks:               eBay Browse API (existing collectors)
  eBay bids (OBO):         eBay Trading API (YOUR listings only — cannot see others')
  eBay transactions:       eBay Finding API completed items (existing collectors)


================================================================================
SECTION 1 — UNIVERSAL COLUMNS
================================================================================
Present on EVERY table, EVERY market. Same names. Same types. No exceptions.

CARD IDENTITY (joins + display):
  id                  SERIAL PRIMARY KEY
  card_id             INTEGER REFERENCES master_card_catalog(id)  -- THE join key
  card_code           TEXT            -- BST-004-1E (display only, never join on this)
  card_name           TEXT
  set_name            TEXT
  card_number         TEXT
  grader              TEXT CHECK (grader IN ('CGC','PSA','BGS','SGC'))
  grade               TEXT            -- verbatim raw_label from master_grade_catalog
  edition             TEXT NOT NULL DEFAULT 'N/A'
  print_variant       TEXT NOT NULL DEFAULT 'Standard'

PLATFORM:
  platform            TEXT NOT NULL   -- 'ebay', 'courtyard', 'collector_crypt'
  currency            TEXT            -- 'USD', 'USDC', 'SOL'

METADATA:
  created_at          TIMESTAMPTZ DEFAULT NOW()
  raw_payload         JSONB           -- full source API response for debugging/enrichment

RULES:
  - card_id is NULLABLE. Rows that fail resolution still insert with card_id = NULL.
    They are excluded from arbitrage signals, not from the table.
  - card_code is populated via JOIN to master_card_catalog, never manually entered.
  - NULL means "unknown/not available." 0 means "zero/free." Never conflate them.
  - platform is CHECK-constrained per table (e.g. courtyard_asks enforces 'courtyard').


================================================================================
SECTION 2 — [MARKET]_ASKS (sell side)
================================================================================
Active + historical listings. Mutable (outcome, price, last_seen_at update over time).

UNIVERSAL COLUMNS (Section 1)
  +

LISTING DETAILS:
  item_id             TEXT UNIQUE     -- eBay item ID / NFT token ID / OpenSea order hash
  listed_price        DECIMAL(10,2)   -- original ask price
  final_price         DECIMAL(10,2)   -- what it actually sold for (NULL if unsold)
  listed_at           TIMESTAMPTZ
  ended_at            TIMESTAMPTZ
  days_listed         INT             -- derived: ended_at - listed_at

OUTCOME (what happened to this listing):
  outcome             TEXT            -- per-market CHECK constraint (see below)

DEMAND SIGNALS:
  watchers            INT             -- eBay only (Courtyard/CC: NULL)
  views               INT             -- NULL on all platforms (no API exposes this)
  bid_count           INT             -- auction bid count or OBO offer count
  best_offer_received DECIMAL(10,2)   -- highest OBO/offer received (eBay own listings only)

RELIST TRACKING:
  relist_count        INT DEFAULT 0
  original_item_id    TEXT            -- first listing by this seller for this card
  previous_ask_price  DECIMAL(10,2)   -- what they asked last time (price reduction tracking)

SELLER:
  seller_username     TEXT            -- eBay username / display name
  seller_address      TEXT            -- wallet address (tokenized only, NULL on eBay)
  seller_feedback     INT             -- eBay only (NULL on tokenized)
  seller_feedback_pct NUMERIC         -- eBay only (NULL on tokenized)

LISTING META:
  listing_url         TEXT
  image_url           TEXT
  condition           TEXT
  transaction_type    TEXT            -- 'AUCTION', 'BIN', 'FIXED'
  last_seen_at        TIMESTAMPTZ     -- collector's most recent observation of this listing

TOKENIZED-ONLY (NULL on eBay):
  blockchain          TEXT            -- 'polygon', 'solana'
  nft_contract        TEXT


OUTCOME CHECK CONSTRAINTS — per market:

  ebay_asks:
    CHECK (outcome IN (
      'active',             -- currently live on eBay
      'sold_full_price',    -- BIN sale at listed price
      'sold_auction',       -- auction closed with winning bidder
      'sold_obo',           -- Best Offer accepted below ask
      'expired_no_bids',    -- listing expired, zero bids/offers received
      'expired_with_bids',  -- listing expired, had bids but reserve not met
      'cancelled_seller',   -- seller pulled the listing voluntarily
      'relisted',           -- seller ended and reposted (new item_id, linked via original_item_id)
      'ended_other'         -- catch-all for uncommon end reasons
    ))

  courtyard_asks:
    CHECK (outcome IN (
      'active',             -- currently listed on Courtyard/OpenSea
      'sold',               -- token purchased by buyer
      'delisted',           -- seller removed listing (no sale)
      'price_reduced',      -- seller lowered ask (tracked via order_price_history)
      'expired'             -- listing had expiration and lapsed without sale
    ))

  collector_crypt_asks:
    CHECK (outcome IN (
      'active',             -- currently listed on CC/Magic Eden
      'sold',               -- token purchased by buyer
      'delisted',           -- seller removed listing
      'price_reduced',      -- seller lowered ask
      'expired',            -- listing lapsed
      'buyback'             -- platform bought at 85-90% FMV (CC-specific mechanism)
    ))


================================================================================
SECTION 3 — [MARKET]_BIDS (buy side)
================================================================================
Open buy offers. Mutable (status changes as offers are accepted/declined/expire).

UNIVERSAL COLUMNS (Section 1)
  +

OFFER DETAILS:
  offer_id            TEXT UNIQUE     -- eBay offer ID / on-chain order hash
  offer_price         DECIMAL(10,2)
  offer_type          TEXT CHECK (offer_type IN (
                        'individual',   -- offer on one specific item/token
                        'collection',   -- offer on ANY token in the collection
                        'trait',        -- offer matching specific grade/set/grader combo
                        'obo'           -- eBay Best Offer (seller-side visibility only)
                      ))

LIFECYCLE:
  offered_at          TIMESTAMPTZ
  expires_at          TIMESTAMPTZ
  is_active           BOOLEAN DEFAULT TRUE
  status              TEXT CHECK (status IN (
                        'open',         -- waiting for seller to respond
                        'accepted',     -- seller accepted → becomes a transaction
                        'declined',     -- seller rejected the offer
                        'expired',      -- offer timed out without response
                        'cancelled'     -- buyer withdrew the offer
                      ))

BUYER:
  buyer_username      TEXT            -- eBay username
  buyer_address       TEXT            -- wallet address (tokenized only)

TOKENIZED-ONLY:
  blockchain          TEXT
  order_hash          TEXT            -- on-chain Seaport/ME order hash

DATA AVAILABILITY BY PLATFORM:
  Courtyard (OpenSea):     FULL — all offer types, all statuses
  Collector Crypt (ME):    FULL — bids endpoint confirmed
  eBay:                    PARTIAL — only YOUR OWN listings' received offers
                           You CANNOT see offers on other sellers' listings.
                           ebay_bids will be sparsely populated.


================================================================================
SECTION 4 — [MARKET]_TRANSACTIONS (completed sales)
================================================================================
Immutable. Once a sale completes, the row never changes.

UNIVERSAL COLUMNS (Section 1)
  +

SALE DETAILS:
  item_id             TEXT UNIQUE     -- same as the originating ask's item_id
  sold_price          DECIMAL(10,2)   -- actual sale price (what buyer paid)
  listed_price        DECIMAL(10,2)   -- what it was listed at (spread analysis: sold vs asked)
  sold_at             TIMESTAMPTZ
  sale_type           TEXT CHECK (sale_type IN (
                        'fixed_price',    -- BIN / direct purchase at listed price
                        'auction',        -- won at auction (may be above or below listed)
                        'offer_accepted', -- OBO / bid accepted (below listed price)
                        'buyback',        -- platform buyback at 85-90% (CC only)
                        'peer_to_peer'    -- direct token transfer with payment (tokenized)
                      ))

PARTIES:
  seller_username     TEXT
  seller_address      TEXT            -- wallet (tokenized only)
  buyer_username      TEXT
  buyer_address       TEXT            -- wallet (tokenized only)

PLATFORM FEES (estimated for third-party sales, actual for YOUR sales):
  platform_fee_pct    DECIMAL(6,4)    -- percentage charged (0.00 / 2.00 / 13.25)
  platform_fee_amt    DECIMAL(10,2)   -- dollar amount
  royalty_fee_pct     DECIMAL(6,4)    -- CC 1% royalty (NULL on others)
  royalty_fee_amt     DECIMAL(10,2)
  payment_processing_pct DECIMAL(6,4) -- eBay 2.9% (NULL on tokenized)
  payment_processing_amt DECIMAL(10,2)

SHIPPING & HANDLING:
  shipping_cost       DECIMAL(10,2)   -- actual or estimated shipping
  insurance_cost      DECIMAL(10,2)
  packaging_cost      DECIMAL(10,2)
  authentication_fee  DECIMAL(10,2)   -- eBay AG $12-22 (NULL on tokenized)

VAULT / REDEMPTION (tokenized only):
  redemption_fee      DECIMAL(10,2)   -- Courtyard $2 handling
  vault_storage_fee   DECIMAL(10,2)
  tokenization_fee    DECIMAL(10,2)   -- cost to mint if physical was vaulted

BLOCKCHAIN (tokenized only):
  gas_fee             DECIMAL(10,6)   -- native token gas (Polygon/Solana — tiny)
  gas_fee_usd         DECIMAL(10,4)   -- converted to USD at time of tx
  blockchain          TEXT
  transaction_hash    TEXT            -- on-chain tx hash (verifiable on-chain)
  nft_contract        TEXT

TAXES:
  sales_tax_collected DECIMAL(10,2)   -- tax charged to buyer
  sales_tax_rate      DECIMAL(6,4)
  income_tax_liability DECIMAL(10,2)  -- estimated tax owed on profit (YOUR sales only)

COST BASIS (populated ONLY on YOUR trades — NULL for observed third-party sales):
  acquisition_price   DECIMAL(10,2)   -- what you paid to acquire this card
  acquisition_market  TEXT            -- where you bought it
  acquisition_date    TIMESTAMPTZ

TOTALS (derived but stored for fast queries — YOUR trades only):
  total_fees          DECIMAL(10,2)   -- sum of all fee fields above
  total_cost_basis    DECIMAL(10,2)   -- acquisition + all fees + shipping + tax
  gross_profit        DECIMAL(10,2)   -- sold_price - acquisition_price
  net_profit          DECIMAL(10,2)   -- sold_price - total_cost_basis
  profit_margin_pct   DECIMAL(6,2)    -- net_profit / sold_price * 100
  roi_pct             DECIMAL(6,2)    -- net_profit / total_cost_basis * 100

MARKET-RATE ESTIMATES (for ALL sales — third-party and yours):
  market_platform_fee_est   DECIMAL(10,2)  -- estimated from fee_schedule
  market_total_costs_est    DECIMAL(10,2)  -- total estimated costs at published rates
  market_net_to_seller_est  DECIMAL(10,2)  -- sold_price - market_total_costs_est

eBay-ONLY:
  shipping            DECIMAL(10,2)   -- eBay shipping charge to buyer
  returns_accepted    BOOLEAN

NOTES ON COST FIELDS:
  - For observed third-party sales: cost basis, total_fees, net_profit = NULL.
    Only market_*_est fields are populated (calculated from fee_schedule).
  - For YOUR trades: both actual AND market_est fields are populated, enabling
    comparison of what you paid vs what the published rate would have been.
  - NULL = unknown/not applicable. 0 = zero/free. Never conflate.

SECTION 5 — COMPLETE TABLE INVENTORY
================================================================================

DIMENSION TABLES (authorities — unchanged):
  master_card_catalog               637 cards / 1,354 variants + card_code + card_id
  master_set_catalog                8 WOTC sets + set_codes (BST/FSL/JGL/TRK/GYH/GYC/WSP/BOG)
  master_grade_catalog              88 grade labels (PSA/CGC/BGS/SGC)
  master_grading_criteria_catalog   4 factors x 4 graders + PSA qualifiers
  seller_profiles                   cross-platform seller dimension

MARKET DATA (per platform — NEW NAMING):
  ebay_bids                         eBay Best Offer / OBO data (YOUR listings only)
  ebay_asks                         eBay listings — active + historical + outcome tracking
  ebay_transactions                 eBay sold comps — full cost basis on your trades

  courtyard_bids                    Courtyard buy offers (OpenSea Offers API)
  courtyard_asks                    Courtyard sell listings (OpenSea Listings API)
  courtyard_transactions            Courtyard completed sales (OpenSea Events API)

  collector_crypt_bids              CC buy offers (Magic Eden Bids API)
  collector_crypt_asks              CC sell listings (Magic Eden Listings API)
  collector_crypt_transactions      CC completed sales (Magic Eden Activity API)

ANALYTICS (see Part 2):
  order_price_history               Bid AND ask price changes across all markets
  fee_schedule                      Platform fee reference table (verified rates)

TRADING (see Part 3):
  tokenized_arbitrage_signals       Cross-market opportunity signals + estimated costs
  arbitrage_positions               Full trade lifecycle + P&L + estimated vs actual

SUPPORT:
  pop_reports                       Graded population counts (CGC/PSA)
  filter_rejections                 n8n workflow rejects
  ebay_watchlist                    Sniper buy-criteria config

SECTION 6 — MIGRATION PATH (old names → new names)
================================================================================

RENAMES:
  ebay_listings                → ebay_asks
                                 + add outcome column
                                 + add demand signal columns (watchers, views, bid_count)
                                 + add relist tracking columns
  sold_listings                → ebay_transactions
                                 + add full cost basis columns
                                 + add market_*_est columns
  courtyard_listings           → courtyard_asks
  courtyard_sales              → courtyard_transactions
  collector_crypt_sales        → collector_crypt_transactions

NEW TABLES:
  ebay_bids                    (new — OBO offer data from your listings)
  courtyard_bids               (new — OpenSea offers API)
  collector_crypt_bids         (new — Magic Eden bids API)
  collector_crypt_asks         (new — Magic Eden listings API)
  order_price_history          (new — see Part 2)
  fee_schedule                 (new — see Part 2)
  arbitrage_positions          (new — see Part 3)

BACK-COMPAT VIEWS (keep alive until ALL consumers migrated):
  CREATE VIEW ebay_listings AS SELECT * FROM ebay_asks;
  CREATE VIEW sold_listings AS SELECT * FROM ebay_transactions;
  CREATE VIEW courtyard_listings AS SELECT * FROM courtyard_asks;
  CREATE VIEW courtyard_sales AS SELECT * FROM courtyard_transactions;

  Views are auto-updatable — INSERT/UPDATE through them hits the real table.
  Drop each view ONLY after confirming zero n8n workflows reference the old name.
  Verify with: grep -r "ebay_listings" n8n-workflows/
