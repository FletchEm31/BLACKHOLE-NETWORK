SECTION 2 — WIREGUARD MESH (THE BACKBONE)
==================================================================

ALL inter-server and server-to-device communication runs over
WireGuard. This is the encrypted backbone of the entire network.

ENCRYPTION STANDARD:
  Protocol:      WireGuard (UDP)
  Cipher:        ChaCha20-Poly1305
  Key exchange:  Curve25519
  Hashing:       BLAKE2s
  PSK:           Additional 32-byte pre-shared key (quantum protection)
  Result:        Military-grade encryption, unreadable to any observer

MESH STRUCTURE:
  LA Hub (wg0) is the central WireGuard server.
  All other nodes/devices connect TO LA as peers.
  No peer connects directly to another peer (hub-and-spoke).

  ┌─────────────────────────────────────────────┐
  │              LA HUB (10.8.0.1)              │
  │              WireGuard Hub (wg0)            │
  └──┬──────────┬──────────┬──────────┬─────────┘
     │          │          │          │
  Frankfurt  Hillsboro    NJ      Devices
  10.9.0.2   10.8.0.6  10.8.0.5  (PC/Phone)

PEER REGISTRY:
  Peer              Public IP         Tunnel IP   PSK      Port
  ─────────────────────────────────────────────────────────────

  NJ US2            <BHN_NJ_PUBLIC_IP>       10.8.0.5    ❌ pend  51820
  Hillsboro US3     <BHN_HIL_PUBLIC_IP>       10.8.0.6    ✅       51821

WHAT AN OBSERVER SEES ON THE WIRE:
  - Random UDP packets between known server IPs
  - No packet content (fully encrypted)
  - No metadata about what's being transferred
  - Packet size and timing only (minimal leakage)
  - Cannot distinguish WireGuard from random UDP

─────────────────────────────────────────────────────────────
3F. DNS RESOLUTION FLOW
─────────────────────────────────────────────────────────────
ALL nodes use dnscrypt-proxy for encrypted DNS.
No plaintext DNS queries ever leave any BHN node.

DNS RESOLVER ROTATION (all nodes — same config):
  dnscrypt-proxy rotates between 6 encrypted resolvers:
  1. Cloudflare      (US)           DoH/DNSCrypt
  2. Quad9           (Global)       DNSCrypt + malware filter
  3. Mullvad         (Sweden)       DNSCrypt — privacy-first
  4. AdGuard         (Cyprus)       DoH — ad blocking
  5. NextDNS         (US)           DoH — customizable
  6. Digitale Gesellschaft (Swiss)  DoH — privacy org

  Selection: fastest responding at query time
  Fallback: automatic if any resolver goes down
  External observer sees: encrypted DNS-over-HTTPS/DNSCrypt
  No observer can see what domains are being resolved

TARGET STATE — Frankfurt DNS Phase 2 (PENDING):
  VPN clients will use Frankfurt as DNS resolver
  Frankfurt dnscrypt-proxy bound to 10.9.0.2:53
  Full tunnel WireGuard profile DNS = 10.9.0.2
  Result: DNS queries appear to originate from Frankfurt (DE)

─────────────────────────────────────────────────────────────
3G. TOR RELAY TRAFFIC FLOW
─────────────────────────────────────────────────────────────
BHN nodes run Tor middle/guard relays (never exit).
Tor relay traffic is independent of VPN client traffic.

  Global Tor network
    → encrypted Tor cells → BHN relay node (ORPort 9001)
    → relay forwards to next Tor hop
    → encrypted Tor cells → out

  BHN sees: encrypted Tor cells only
  Cannot see: origin, destination, or content
  Purpose: contributes bandwidth to Tor network,
           adds cover traffic to nodes,
           supports privacy ecosystem

SECTION 6 — SECURITY LAYERS (ALL NODES)
==================================================================

EVERY BHN NODE HAS:
  Layer 1: UFW (default deny in/out — whitelist only)
  Layer 2: fail2ban (auto-ban brute force attempts)
  Layer 3: CrowdSec (AI-driven threat detection, shared intel)
  Layer 4: Suricata (deep packet IDS/IPS)
  Layer 5: dnscrypt-proxy (encrypted DNS — no plaintext leakage)
  Layer 6: SSH key-only auth (no password auth)
  Layer 7: WireGuard (all inter-node traffic encrypted)

LA HUB ADDITIONAL:
  Layer 8: LUKS2 storage encryption (NVMe + HDD)
  Layer 9: PostgreSQL access controls (role-based)
  Layer 10: WireGuard PSK (quantum-resistant key exchange)

OBFUSCATION LAYERS (exit nodes — Frankfurt, Hillsboro):
  Shadowsocks: disguises VPN traffic as HTTPS
  dnscrypt: disguises DNS queries as HTTPS
  Tor relay: cover traffic blends with relay traffic
  tinyproxy: LA API calls appear from Hillsboro IP
