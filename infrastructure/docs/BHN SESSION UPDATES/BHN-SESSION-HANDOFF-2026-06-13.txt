Task 10: Edge calculation bug fixed ✅ CRITICAL
  Bug: edge was stored as YES-perspective (calibrated_prob - market_implied_prob) for ALL
  contracts. BET_NO rows had negative stored edge, the opposite of reality.
  Confirmed by Q20: "Strong Negative Edge" contracts were winning at 75%.

  Fix in scripts/trading/weather_edge_calculator.py:
    edge_yes = raw_prob - mip                           # trading decision only
    stored_edge = (mip - raw_prob) if action == "BET_NO" else edge_yes
  BET_NO stored_edge is now: (1 - calibrated_prob) - (1 - mip) = mip - calibrated_prob
  Always positive when BHN has genuine NO edge.

  Deployed: scp to /opt/bhn/trading/weather_edge_calculator.py
