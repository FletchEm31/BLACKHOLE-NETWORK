--- POSTGRESQL BOOT RACE FIX (Jun 26) ---

  PostgreSQL was failing to start at boot because the LUKS2 NVMe
  (/mnt/eh-nvme-hot) mounts after systemd starts PG at boot time.
  Fix: added systemd drop-in on LA:
    /etc/systemd/system/postgresql@14-main.service.d/nvme-mount.conf
    [Unit]
    After=mnt-eh\x2dnvme\x2dhot.mount
    Requires=mnt-eh\x2dnvme\x2dhot.mount
  Permanent fix — survives reboots.

--- NBM FETCH FIX (Jun 26) ---

  fetch_nbm() was returning 0 rows because it queried
  `probabilisticQuantileForecast.temperature` from NWS /gridpoints API
  — this field does NOT exist in the NWS API.
  Fix: switched to Open-Meteo GFS ensemble API (ensemble-api.open-meteo.com).
  Requests 28-30 member forecasts, computes daily max per member,
  derives P10/P25/P50/P75/P90 via linear interpolation.
  Stored under nws_office='open_meteo_gfs'. 56 rows inserted on first run.
  Commit: f43a00a

--- KALSHI FIELD RENAME FIX (Jun 26) ---

  Kalshi API v2 renamed all price/volume fields:
    yes_bid → yes_bid_dollars (fractional 0-1 string)
    yes_ask → yes_ask_dollars, no_bid → no_bid_dollars, no_ask → no_ask_dollars
    volume_24h → volume_24h_fp (float string)
    volume → volume_fp, open_interest → open_interest_fp
  Fix: updated weather_data_collector.py to read new _dollars/_fp fields
  with fallback to old names for backwards compat.
  _cents_to_frac() already handles both formats (detects >1 = integer cents,
  ≤1 = already fractional).
  Commit: 415a2d3

SECTION 9 — TRADING STRATEGY SUMMARY
==================================================================

CORE STRATEGY: Boundary Split + Tail No (Iron Condor equivalent)

Step 1: Read NWS hourly forecast for target station
Step 2: If forecast on boundary between two buckets — buy Yes on both
Step 3: Load No contracts on statistically impossible tail buckets
Step 4: Hold to settlement

POSITION SIZING: Half Kelly with liquidity cap
  - Never more than 10% of open interest
  - Never more than 5% of daily volume
  - Spread > 20¢ = skip

MIAMI-SPECIFIC NUANCES:
  - Afternoon thunderstorms (2-5pm) suppress daily high by 2-4°F
  - Onshore sea breeze (SE/S wind > 5mph) suppresses high
  - High humidity + dewpoint spread limits afternoon heating
