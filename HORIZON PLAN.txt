MEMORY & DATA ARCHITECTURE
================================================================================

THREE MEMORY LANES (all in PostgreSQL pgvector):
  conversation    → every chat/call turn, compressed after 90 days, kept forever
  security_event  → anything triggering alerts, kept forever (audit trail)
  market_data     → eBay + financial signals, 1 year rolling

RETENTION POLICY:
  | Data Type          | Hot (NVMe)  | Cold (HDD)    | Purge       |
  | Conversations      | 90 days     | Forever       | Never       |
  | Security events    | 90 days     | Forever       | Never       |
  | Market data        | 90 days     | 1 year        | After 1yr   |
  | Call transcripts   | 90 days     | 1 year        | After 1yr   |
  | Raw call audio     | Deleted immediately after transcription      |
  | News/weather cache | 7 days      | Not archived  | After 7days |

QUERY ARCHITECTURE (cost efficiency — RAG-first):
  1. Query local PostgreSQL first (free)
  2. pgvector similarity search (free)
  3. Only hit Claude API if local data insufficient
  4. Cache frequent Q&A pairs in PG (free)
  5. Haiku router for query classification (~$0.50/month)
  6. Nightly digest summarization (Pulse already building this)
